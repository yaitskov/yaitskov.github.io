data YamlPostgresDefaults #

A set of defaults that's useful to capture from the user - allows us to run SQL ambiguous in database/schema, or run without asking an explicit target server from the user.

We may also want to bake in optional defaults on username, but for now I'm keeping that separate to be pulled in from authentication info meant to reside separately from the spec config.

For now, we're keeping all Postgres based cases unified. We may break them out as needed in the future.
